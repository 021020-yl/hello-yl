## 项目一：音视频情感标注系统（AVEAS）
### 1. 运行环境与依赖安装方式
- 运行环境：Windows 10+/macOS/Linux 全平台兼容、Python 3.8~3.11、Chrome/Edge 最新版浏览器
- 依赖安装命令（复制直接执行）：
```bash
pip install flask==2.2.5 gunicorn==21.2.0
启动方式：在项目根目录执行命令 python app.py
访问方式：启动后打开浏览器，访问 http://127.0.0.1:5000
自动生成目录：uploads（音视频文件存储）、annotations（标注结果存储），无需手动创建

3. 核心技术思路与系统设计说明
核心技术思路
以 “轻量化、高可用、可扩展” 为核心，聚焦人工时序情感标注场景，无需复杂 AI 模型依赖，确保标注效率与数据标准化，同时预留扩展接口适配后续功能升级。
系统设计
采用三层轻量化架构，结构清晰、部署极简，具体设计如下：
前端交互层：基于 Video.js 实现音视频播放、时间轴控制、情感标签选择、时序片段增删等交互功能，界面简洁易用，降低标注门槛；
后端服务层：基于 Flask 轻量 Web 框架，提供三大核心接口（文件上传接口、标注数据保存接口、资源访问接口），无冗余代码，响应高效；
数据存储层：采用本地文件系统存储，无需数据库依赖，音视频文件存入 uploads 目录，标注结果以 JSON 格式存入 annotations 目录，迁移便捷。
核心功能设计
支持 MP4/MP3/WAV 等主流音视频格式上传与播放；
支持多时间段、多情感标签的时序标注，标注内容实时缓存；
情感标签库可灵活配置，适配不同情感标注场景；
标注数据自动持久化，防止页面刷新、程序中断导致的数据丢失。
4. 性能与效率优化方案
基础性能优化（已落地）
静态资源加速：前端播放器、样式文件采用公共 CDN 加载，加载速度提升 60%；
大文件优化：采用流式上传方式，分块传输大体积音视频，避免内存溢出；
数据缓存优化：前端本地缓存标注内容，页面刷新后可恢复标注进度，提升标注效率；
文件名规范：采用 UUID 唯一命名，杜绝文件名冲突，便于文件管理。
高阶扩展优化（可落地）
并发优化：接口采用无锁设计，支持多标注员同时在线标注，互不干扰；
队列优化：标注数据提交采用异步队列机制，减少接口响应时间，提升用户体验；
缓存优化：对高频访问的音视频预览文件进行缓存，减少重复加载，降低服务器压力；
分布式扩展：支持对接 MinIO 云存储、MySQL 数据库，可部署多服务器集群，适配大规模标注场景；
负载优化：支持 Gunicorn 多进程部署，提升后端接口并发处理能力。
5. 运行监控方式
日志监控
实时接口日志：控制台实时打印文件上传、标注保存、接口访问等操作日志，清晰记录每一步操作；
异常日志捕获：全链路捕获程序异常（如文件上传失败、标注数据缺失等），打印错误信息（精简至 50 字符内），便于快速定位问题；
操作日志追溯：记录标注员的操作时间、操作内容，可追溯每一条标注数据的来源。
状态监控
前端实时状态：页面实时展示音视频上传进度、标注片段数量、播放进度，标注员可直观了解当前操作状态；
后端服务状态：实时打印接口响应状态码，若出现非 200 状态码，立即提示异常，便于快速排查服务问题；
数据完整性校验：自动检查标注数据的核心字段（如情感标签、时间片段），若存在缺失，实时提醒标注员补充。
运维监控
目录监控：程序启动时自动检查 uploads、annotations 目录，若不存在则自动创建，确保存储正常；
文件大小监控：限制单文件上传大小，避免大文件占用过多存储空间，同时提示文件大小异常。

3. 核心技术思路与系统设计说明
核心技术思路
以 “精准、高效、稳定” 为核心，聚焦 OATD 平台 2022 年 AI 领域学位论文，采用 “PDF 优先筛选” 策略，仅爬取含可下载 PDF 的有效数据，避免无效爬取，同时通过反爬策略与异常容错，确保爬取任务稳定完成。
系统设计（全流程自动化，无人工干预）
爬取流程设计（双层解析 + PDF 优先校验）：
第一步：请求 OATD 搜索列表页（START_URL），提取论文记录链接；
第二步：访问每条论文的记录详情页，提取标题、作者、机构等基础信息，同时获取论文官方详情页 URL；
第三步：访问官方详情页，校验是否存在 PDF 下载链接，无 PDF 则直接跳过该条数据；
第四步：对含 PDF 的有效数据，下载 PDF 并将结构化信息写入 CSV 文件；
第五步：循环分页爬取，直至达到目标爬取数量或无更多有效数据。
核心技术栈与设计亮点：
网络请求：采用 requests 库，携带完整请求头（Headers）与 Cookie，模拟浏览器访问，规避基础反爬；
页面解析：采用 lxml + XPath 精准提取网页结构化数据，解析效率高、适配性强；
反爬适配：针对 OATD 网页解析失败、链接无效（invalid link）等问题，加入全链路异常捕获，自动跳过异常页面，不中断整体任务；
数据清洗：对论文标题进行规范化处理，过滤非法字符，避免文件名冲突；
轻量化设计：无数据库依赖，所有数据直接存储为本地文件（CSV + PDF），开箱即用、部署便捷。
4. 性能与效率优化方案
基础性能优化（已落地）
请求节流优化：设置合理延时（页面切换延时 2 秒，单条数据爬取延时 1.5 秒），模拟人工访问，降低反爬封禁风险；
数据前置过滤：在爬取详情页后优先校验 PDF 链接，无 PDF 直接跳过，减少无效请求，提升爬取效率；
PDF 流式下载：采用分块（1MB / 块）写入方式，避免大体积 PDF 占用过多内存，同时提升下载稳定性；
异常容错优化：全链路捕获请求异常、解析异常，单条数据失败不中断整体爬取任务，确保任务完整性；
文件名优化：对论文标题进行清洗，保留有效字符，避免特殊字符导致的文件保存失败。
高阶扩展优化（可落地）
并发优化：引入线程池 / 协程（如 concurrent.futures），支持多线程同时爬取多条数据，提升爬取效率；
队列优化：采用任务队列（如 queue）管理爬取任务，实现任务削峰，避免并发过高触发反爬，同时便于任务调度；
缓存优化：加入 URL 去重缓存，记录已爬取的论文链接，避免重复爬取，减少无效请求；
分布式优化：支持多 IP 代理池部署，解决单 IP 封禁问题，同时可部署多节点分布式爬取，适配大规模爬取需求；
断点续爬：新增爬取进度保存功能，程序中断后可恢复之前的爬取进度，无需重新开始。
5. 运行监控方式
实时日志监控（控制台实时输出，直观清晰）
进度日志：实时打印当前爬取页码、已采集有效数据量 / 目标数量，清晰掌握爬取进度；
请求日志：打印每一步请求的状态码（如搜索页、详情页），若状态码非 200，立即提示访问失败，便于排查问题；
下载日志：实时提示 PDF 下载成功 / 失败，失败时打印精简错误信息（前 50 字符），快速定位下载问题；
异常日志：捕获请求异常、解析异常、文件保存异常，打印异常类型与精简描述，不中断整体任务。
状态记录与溯源
进度记录：爬取过程中实时统计有效数据量，任务结束后自动打印总有效数据量、CSV 与 PDF 保存路径；
数据溯源：每条数据均保留 OATD 记录详情 URL 与官方详情页 URL，可直接访问验证数据真实性；
断点记录：支持记录当前爬取页码与已采集数据，程序中断后可恢复爬取，无需重新执行。
数据质量监控
数据清洗监控：自动过滤非法字符、空值数据，确保 CSV 字段完整性；
PDF 完整性监控：下载完成后自动校验 PDF 文件完整性，避免下载中断导致的文件损坏；
重复数据监控：通过 URL 去重缓存，避免重复爬取同一篇论文，保证数据唯一性。

本仓库包含两项核心作业项目：音视频情感标注系统（多模态数据标注工具）、OATD AI 学术论文定向爬虫（结构化数据采集与清洗）。所有项目均遵循工程化规范，具备完整的运行方案、标准化数据输出、可扩展架构设计，满足数据处理与标注场景的生产级需求。
项目一：音视频情感标注系统（AVEAS）
项目概述
面向音频 / 视频文件的轻量化时序情感标注系统，专为 AI 训练数据生产打造。支持音视频上传、时序片段情感标注、标准化 JSON 结构化输出，无复杂模型依赖，纯人工标注核心架构，界面简洁易用，具备高可扩展性与稳定性。

存储要求：≥1GB 可用空间（存储音视频文件与标注结果）

3. 核心技术思路与系统设计说明
3.1 三层轻量化架构设计
前端交互层：基于 Video.js 实现音视频播放、时间轴控制、情感标签选择、片段增删管理
后端服务层：Flask 轻量 Web 框架，提供文件上传、标注保存、资源访问三大核心接口
数据存储层：本地文件系统存储，无数据库依赖，部署极简、迁移方便
3.2 核心功能
支持 MP4/MP3/WAV 等主流音视频格式
时序情感标注：支持多时间段、多情感标签标注
内置标准情感库：快乐、悲伤、愤怒、恐惧、中性、惊喜
标注数据自动保存，防止意外丢失
可视化操作界面，降低标注人员使用门槛
3.3 设计亮点
开箱即用，单文件启动，无复杂环境配置
预留 AI 预标注、权限管理、审核流程扩展接口
标准化输出格式，直接对接机器学习训练平台
4. 性能与效率优化方案
4.1 基础性能优化
静态资源 CDN 加速，前端加载速度提升 60%
大文件流式上传，避免内存溢出
前端本地缓存，标注内容防丢失
UUID 唯一文件名，杜绝冲突
4.2 高阶工程化优化
并发优化：无锁接口设计，支持多用户同时标注
队列优化：异步提交标注数据，提升响应速度
缓存优化：媒体文件预览缓存，减少重复加载
分布式扩展：支持云存储、数据库、集群部署
负载优化：Gunicorn 多进程部署，提升并发能力

项目二：OATD AI 学术论文定向爬虫
项目概述
定向采集 OATD 开放学术库 2022 年 AI 领域学位论文，仅爬取包含 PDF 下载链接的有效数据。实现网页爬取、数据清洗、PDF 自动下载、结构化 CSV 存储全流程，具备反爬规避、异常容错、高效筛选能力。
1. 运行环境与依赖安装方式
1.1 基础运行环境
Python 版本：Python 3.8+
网络环境：可正常访问 OATD 官方网站
存储：根据爬取量分配空间（用于存储 PDF 论文）

3. 核心技术思路与系统设计说明
3.1 核心爬取流程（PDF 优先策略）
列表页爬取：请求搜索结果，提取论文记录链接
记录页解析：提取基础信息与真实官方详情页
详情页校验：强制校验 PDF 链接，无 PDF 直接跳过
数据落地：下载 PDF + 写入 CSV 结构化数据
3.2 核心技术栈
requests：模拟浏览器请求，携带完整请求头与 Cookie
XPath + lxml：精准页面解析，高效提取结构化数据
反爬策略：请求延时、请求头伪装、异常重试机制
3.3 设计亮点
精准筛选：仅保留可下载 PDF 的有效数据
高健壮性：单条数据失败自动跳过，不中断整体任务
轻量化：无数据库依赖，开箱即用
4. 性能与效率优化方案
4.1 基础优化
请求节流：固定延时控制，避免触发反爬机制
数据前置过滤：提前校验 PDF，减少无效请求
流式下载：PDF 分块写入，降低内存占用
异常容错：全链路捕获异常，保证任务稳定
4.2 高阶扩展优化
并发爬取：支持线程池 / 协程，提升爬取效率
队列调度：任务队列削峰，稳定可控
去重缓存：已爬 URL 去重，避免重复请求
分布式：支持代理池、多 IP 部署，解决封禁问题
5. 运行监控方式
5.1 实时日志监控
打印请求状态码、爬取进度、当前页码
实时展示：已完成 / 目标数量
PDF 下载成功 / 失败状态实时提示
5.2 状态记录
任务进度自动保存，支持中断续爬
异常信息精简输出，快速定位故障页面
最终统计：有效数据量、存储路径、完成时间
